## 脆弱性の報告

Vulflare のセキュリティ脆弱性を発見した場合は、**公開 Issue には報告しないでください**。

代わりに、GitHub の **[Private Security Advisories](https://github.com/ikech4n/vulflare/security/advisories/new)** を使用して非公開で報告してください。

### 報告に含める内容

- 脆弱性の説明と影響範囲
- 再現手順（できるだけ詳細に）
- 影響を受けるコンポーネント（Worker API / Web / 認証 / DB など）
- 可能であれば PoC（概念実証コード）
- 推奨される修正方法（任意）

## 対応タイムライン

| フェーズ | 目標期間 |
|---------|---------|
| 報告の確認 | 48時間以内 |
| 初期評価・深刻度判定 | 7日以内 |
| 修正リリース | 重大度に応じて変動（目安：30日以内） |
| 公開開示 | 修正リリース後 |

## スコープ

以下はセキュリティ報告の対象です：

- 認証・認可バイパス
- SQLインジェクション
- XSS（クロスサイトスクリプティング）
- CSRF（クロスサイトリクエストフォージェリ）
- 機密データの漏洩
- セッション管理の欠陥
- Cloudflare Workers / D1 / KV に関連するセキュリティ問題

以下は対象外です：

- デモ環境・テスト環境での問題
- ユーザーが自己設定した誤設定
- サードパーティサービス（Cloudflare 自体のインフラなど）の脆弱性
- ベストプラクティスの提案（セキュリティ上の問題を伴わないもの）
